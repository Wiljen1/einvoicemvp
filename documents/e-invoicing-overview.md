# Sample Domain Notes: E-Invoicing

This is sample content for local indexing tests. It is not used by the production UI as a hardcoded domain assumption.

The E-Invoice MVP is intended to help users answer questions about electronic invoicing workflows using only the active approved document source.

The MVP should keep answers short, use business-friendly language, include source references, and clearly state when information is missing from the approved document set.

The application should show Codex availability, active document-source status, indexed file counts, editable guardrails, and a chat interface.

For the local MVP, the approved source can be a local folder, a OneDrive-synced SharePoint folder, or manually uploaded demo documents.
